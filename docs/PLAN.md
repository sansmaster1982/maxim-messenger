# План: APK + дизайн как у оригинального MAX

## Цель

Собрать рабочий `app-debug.apk` для Android, переделать UI под фирменный стиль MAX (oneme.ru) и закрыть пробелы по функционалу относительно оригинала.

## Этап A — собрать APK любой ценой

Текущий блокер: в bash-окружении Claude Code JDK 17 падает на NIO Pipe (Unix Domain Sockets) при запуске Gradle. Перечень попыток обхода:

1. **A1** Подсунуть Gradle JBR от Android Studio (обычно JDK 21+, fallback корректнее). Прописать `org.gradle.java.home` в `android/gradle.properties`.
2. **A2** Прогнать билд с `--no-daemon --no-parallel --no-configuration-cache` и `org.gradle.workers.max=1` (если A1 не сработал).
3. **A3** Запуск через WSL (если стоит) — там UDS работает нормально.
4. **A4** Запуск через `cmd.exe /S /C "gradlew assembleDebug"` отдельным процессом с чистым environment.
5. **A5** В крайнем случае — chmod artefact: собрать через user action в Android Studio Build → Build APK.

Каждая попытка → лог + следующий шаг. Записываю в `PROGRESS.md`.

## Этап B — дизайн под оригинальный MAX

Что есть у оригинала (по декомпилу APK и публичным материалам oneme.ru):
- Primary blue `#2F6FED`, светлый/тёмный темплейт Material 3.
- Bottom navigation: **Чаты / Звонки / Контакты / Настройки** (4 таба).
- На экране чатов: поиск сверху + FAB «новый чат».
- В чате: статус «онлайн / был в …», тапнул заголовок → профиль собеседника.
- Список чатов с группированием «закреплённое», аватары круглые, индикатор непрочитанных синий круг с числом.

Шаги:
1. **B1** `MainShell` с `BottomNavigationBar`: четыре таба, IndexedStack чтобы сохранять state. Чаты, Звонки, Контакты, Настройки.
2. **B2** Тонкий редизайн `ChatsListScreen`: поиск-bar поверх списка, FAB «новый чат», вид tile близко к оригиналу.
3. **B3** Экран «Звонки» — список истории звонков (пока заглушка из локальной таблицы; функция звонков сама — TODO, WebRTC выходит за объём).
4. **B4** В `ChatScreen` — клик по AppBar.title открывает `ProfileScreen` с инфо собеседника / группы.
5. **B5** Тема: уточнить scheme под MAX-палитру, шрифты SF Pro Display / Roboto, скругления и расстояния.
6. **B6** Иконка приложения MAX (адаптивный mipmap) — взять SVG-логотип из открытых источников или нарисовать заглушку «M».

## Этап C — функции из оригинала

В оригинальном MAX есть, у нас пока нет:
- Звонки голос/видео (WebRTC) — UI-каркас сделаем, runtime отложим.
- Профиль собеседника (`B4`).
- Архив чатов — long-press → «архивировать», экран архива.
- Закреплённые сообщения в чате.
- Поиск по сообщениям внутри чата.
- Тайпинг-индикатор от собеседника (приходит push? проверить).
- Reactions на сообщения.
- Forwarding.
- Pin chat.

Реализую B-шаги первым приоритетом + Профиль (`C-profile`), Архив (`C-archive`), Pin (`C-pin`), Forward (`C-fwd`), Reactions (`C-react`). Звонки/типинг inbound оставлю с TODO.

## Этап D — финальная сборка APK

После A+B+C прогнать analyze + тесты + ещё одну сборку APK. Положить в `build/app/outputs/flutter-apk/app-debug.apk`.
